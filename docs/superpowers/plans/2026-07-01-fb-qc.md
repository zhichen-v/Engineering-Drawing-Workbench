# FB_QC Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add FB_QC as a third Excel export format without changing existing MIP or QC behavior.

**Architecture:** Reuse the current QC parser, writer, API endpoint, and React export UI. Add one FB_QC layout variant, select the new template and filenames through `fill_QC.py --format FB_QC`, and embed QC's existing symbol lookup worksheet in the new workbook.

**Tech Stack:** Python 3.12, FastAPI/Pydantic, pytest, Excel COM/pywin32, React 19, Vite 7.

        text=True,
        encoding="utf-8",
        check=False,
    )
